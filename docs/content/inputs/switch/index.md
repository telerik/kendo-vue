# Kendo UI for Vue Native Switch Overview

The Kendo UI for Vue Native Switch lets the user toggle between checked and unchecked states.

The Kendo UI for Vue Switch component is part of the Kendo UI for Vue library of Vue components. It is distributed through NPM under the [kendo-vue-inputs package](https://www.npmjs.com/package/@progress/kendo-vue-inputs).

## Key Features

* [Disabled Switch]({% slug disabled_switch %})&mdash;Render a disabled Switch and keep it disabled until your requirements are fulfilled.
* [Controlled mode]({% slug controlled_switch %})&mdash;Take advantage of the built-in options that allow you to control the value of the Switch.
* [Default state]({% slug default_state %})&mdash;The Switch allows you to set its default value.
* [Labels]({% slug labels_switch %})&mdash;You can set strings as labels for the Switch.
* [Globalization]({% slug globalization_inputs %})&mdash;The Kendo UI for Vue Switch comes with globalization features that allow you to create applications that can be used all over the world.
* [Keyboard navigation]({% slug keyboard_navigation_switch %})&mdash;The Switch allows end-users to control the state of the component with the `space` and `enter` keys.
support many keyboard shortcuts that allow users to interact with the component.
* [Appearance]({% slug appearance_switch %}) - Configure the appearance of the Switch.
* [Accessibility]({% slug accessibility_switch %})&mdash;The Switch component is accessible for screen readers and provides full WAI-ARIA support.

>To learn more about the appearance, anatomy, and accessibility of the Switch, visit the [Progress Design System documentation](https://www.telerik.com/design-system/docs/components/switch/). This information portal offers rich component usage guidelines, descriptions of the available style variables, and globalization support details.
